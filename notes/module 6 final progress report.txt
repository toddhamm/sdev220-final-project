Here is an update on my final project for SDEV 220. 

Currently, I have selected the Custom TKinter module for the graphical 
user interface (GUI). I like it better than the stock TKinter module. 
It reminds me a lot of the Bootstrap framework for HTML and CSS (websites) and is 
built on top of TKinter. 

The documentation for Custom TKinter is https://customtkinter.tomschimansky.com/tutorial/ 
and is very useful. I looked at a few other GUI modules, but I liked Custom TKinter 
the best and it is easy to use. 

Beyond that, I have a demo program built that can be tested if desired, or take a 
look at the screen shots. The demo shows the basic functions (buttons at the top) and 
it imports the sample csv using a file dialog. 

I have a lot to do, obviously, but I believe I can complete the project as outlined in 
my original specifications without any issues and it the time remaining. 

What I will be working on next is the core functionality of the app:
- parse the csv file, process the contents and output to the screen the results noted 
in the original scope of work (attendance / absence trends for a grade level at a school)
- store the report in a database or as a csv and pdf file so that it can be accessed 
later
- add a list view of stored reports
- provide a delete button to delete stored reports
- line the buttons up horizontally
- separate the logic into classes and a main program
